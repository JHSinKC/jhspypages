What is the __file__ constant?
When a module is loaded in Python, __file__ is set to its name. Use 'os' module to find the directory containing the file.
Find 'current working directory' (cwd).
[python console] os.getcwd() # returns a string representing the current working directory
Find the 'real pathname' of the current working directory.
[python console] os.path.realpath(path) # returns "the canonical path of the specified filename, eliminating any symbolic links encountered in the path"
Find the 'real pathname' of the current working directory.
[python console] os.path.dirname(path) # returns "the directory name of pathname path"
Find the 'real pathname' of the current working directory.
[python console] os.chdir(path) ("change the current working directory to path")
How to change the "current working directory" (cwd).
[python console] import os The os module comes with Python; you can import it anytime, anywhere.
When you run the graphical Python Shell, the current working directory starts as the directory where the Python Shell executable is. On Windows the default directory is c:\Python37. If you run the Python Shell from the command line, the current working directory starts as the directory you were in when you call python3.
[python console] os.getcwd() to display the current working directory.
[python console] os.chdir() to change the current working directory.

17.2 'multiprocessing' - Process-based parallelism
Version: {} Released: 2018 no warnings
multiprocessing is a package that supports spawning processes using an API similar to the threading module. The multiprocessing package offers both local and remote concurrency, effectively side-stepping the Global Interpreter Lock by using subprocesses instead of threads.
'multiprocessing' supports spawning processes using an API similar to the threading module; offering both local and remote concurrency. It effectively side-steps the Global Interpreter Lock by using subprocesses instead of threads; allows the programmer to fully leverage multiple processors on a given machine.
The multiprocessing module introduces APIs which do not have analogs in the threading module. The Pool object which offers a convenient means of parallelizing the execution of a function across multiple input values, distributes the input data across processes (data parallelism).
